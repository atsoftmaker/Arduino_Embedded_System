# Lab 04: การควบคุม Digital Output (LED & Relay 220V)

## Overview (ภาพรวม)
แลปนี้เป็นจุดเปลี่ยนจาก "การอ่านค่า (Input)" ไปสู่ "การสั่งงาน (Output)" ผู้เรียนจะใช้คำสั่ง `digitalWrite()` เพื่อสั่งเปิด-ปิด LED และต่อยอดไปควบคุม **โมดูล Relay** ซึ่งทำหน้าที่เป็นสวิตช์อิเล็กทรอนิกส์ตัดต่อวงจรไฟฟ้ากำลังสูง (เช่น หลอดไฟ 220V, ปั๊ม, มอเตอร์) ได้อย่างปลอดภัยโดยที่บอร์ดยังทำงานที่ 5V

แนวคิดสำคัญคือ Relay แยกวงจรควบคุม (5V Logic) ออกจากวงจรกำลัง (220V) ด้วยขดลวดแม่เหล็กไฟฟ้าและหน้าสัมผัส (Contact) ทำให้ไมโครคอนโทรลเลอร์สั่งงานอุปกรณ์ในโรงงานได้

## ⚠️ คำเตือนด้านความปลอดภัย
> งานส่วนไฟ 220V (AC) ต้องทำภายใต้การดูแลของวิทยากรเท่านั้น **ตัดเบรกเกอร์/ถอดปลั๊กก่อนต่อสายฝั่งโหลดทุกครั้ง** ห้ามจับขั้ว COM/NO/NC ขณะมีไฟเข้า สำหรับการฝึกทั่วไป แนะนำให้ทดสอบด้วยหลอดไฟ LED 5V ก่อน แล้วจึงสาธิตฝั่ง 220V โดยวิทยากร

## Hardware Wiring (การต่อวงจร)

**ฝั่งควบคุม (Logic):**

| Relay Module | Arduino UNO |
| :--- | :--- |
| **VCC** | 5V |
| **GND** | GND |
| **IN1** (Signal) | **D8** |

**ฝั่งโหลด 220V (Contact ของ Relay):**

| ขั้ว Relay | การต่อ |
| :--- | :--- |
| **COM** (Common) | สายไฟ Line (L) จากแหล่งจ่าย |
| **NO** (Normally Open) | ต่อไปยังอุปกรณ์โหลด (เช่น หลอดไฟ) |
| **NC** (Normally Closed) | ไม่ใช้ในแลปนี้ |

> หมายเหตุ: Relay Module ส่วนใหญ่เป็นแบบ **Active LOW** (ส่ง LOW = Relay ทำงาน/ON) ให้สังเกตไฟ LED บนโมดูล ถ้าตรรกะกลับด้านให้สลับ HIGH/LOW ในโค้ด

## Code
```cpp
// Lab 04: ควบคุม Relay เปิด-ปิดโหลดเป็นจังหวะ
const int relayPin = 8;

// Relay Module ส่วนใหญ่เป็น Active LOW
// ปรับสองค่านี้ให้ตรงกับโมดูลที่ใช้
const int RELAY_ON  = LOW;
const int RELAY_OFF = HIGH;

void setup() {
  Serial.begin(9600);
  pinMode(relayPin, OUTPUT);
  digitalWrite(relayPin, RELAY_OFF); // เริ่มต้นให้ปิดไว้ก่อนเพื่อความปลอดภัย
  Serial.println("--- Lab 04: Relay Control ---");
}

void loop() {
  Serial.println("สั่ง Relay: ON (โหลดทำงาน)");
  digitalWrite(relayPin, RELAY_ON);
  delay(3000);   // ติด 3 วินาที

  Serial.println("สั่ง Relay: OFF (ตัดโหลด)");
  digitalWrite(relayPin, RELAY_OFF);
  delay(3000);   // ดับ 3 วินาที
}
```

## คำอธิบายโค้ด
- `RELAY_ON / RELAY_OFF` แยกตัวแปรไว้เพื่อรองรับ Relay ทั้งแบบ Active LOW และ Active HIGH โดยแก้ที่เดียว
- ตั้งค่าเริ่มต้นเป็น `RELAY_OFF` ใน `setup()` เพื่อป้องกันโหลดทำงานทันทีตอนบอร์ดบูต
- `delay()` ใช้คุมจังหวะ แต่ในงานจริงควรหลีกเลี่ยง (ดู Lab 5.1 เรื่อง `millis()`)
