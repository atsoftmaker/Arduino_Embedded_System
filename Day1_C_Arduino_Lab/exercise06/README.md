# Exercise 06: ลูกเล่นการแสดงผลตัวเลข (Print Formatting)

ในแบบฝึกหัดนี้ ผู้เรียนจะได้เรียนรู้วิธีสั่งให้บอร์ด Arduino แสดงผลตัวเลขประเภทเดียวกันในหลายมิติ เช่น แสดงตัวเลขในฐานเลขที่แตกต่างกัน หรือควบคุมทศนิยมว่าจะให้พิมพ์ออกมากี่ตำแหน่ง

---

## 💡 แนวคิดเข้าใจง่าย (Analogy)

ให้คิดว่า **"ตัวเลขเดียวกัน"** เปรียบเสมือน **"แอปเปิ้ล 1 ผล"**
แต่เราสามารถเรียกหรือมองแอปเปิ้ลผลนี้ในหลายภาษาได้:

1. **การปรับฐานตัวเลข (Number Bases)**
   * เรียกแบบภาษาไทย (ฐานสิบ / DEC) : ได้คำว่า **"255"** (ฐานสิบที่เราใช้ในชีวิตประจำวันทั่วไป)
   * เรียกแบบภาษาอังกฤษ (ฐานสิบหก / HEX) : ได้คำว่า **"FF"** (นิยมใช้ระบุค่าสี เช่น #FFFFFF หรือข้อมูลสื่อสาร)
   * เรียกแบบภาษาญี่ปุ่น (ฐานสอง / BIN) : ได้คำว่า **"11111111"** (เป็นเลขฐานหลักที่ระบบคอมพิวเตอร์และขาพอร์ตไอซีมองเห็น)

2. **การปรับจุดทศนิยม (Decimal Places)**
   * เปรียบเหมือนการซูมกล้องส่องวัตถุ หากเรามองผ่านๆ (Default) อาจจะเห็นทศนิยมแค่ 2 ตำแหน่ง (`3.14`) แต่หากปรับความละเอียดให้สูงขึ้น ก็สามารถมองเห็นค่าละเอียดได้ถึง 4 ตำแหน่ง (`3.1416`) หรือ 6 ตำแหน่ง (`3.141593`) ตามที่เราต้องการ

---

## 📊 ตารางสรุปรูปแบบการแสดงผลของค่า `255`

| รูปแบบการแสดงผล | คำสั่งที่ใช้ | ผลลัพธ์ที่พิมพ์ออกมา |
| :--- | :--- | :--- |
| **ฐานสิบ (Decimal)** | `Serial.println(colorCode, DEC)` | `255` |
| **ฐานสิบหก (Hexadecimal)** | `Serial.println(colorCode, HEX)` | `FF` |
| **ฐานสอง (Binary)** | `Serial.println(colorCode, BIN)` | `11111111` |

---

## 🔍 อธิบายโค้ดที่สำคัญ

* **`Serial.println(ตัวแปร, DEC/HEX/BIN);`**
  พารามิเตอร์ตัวที่สองจะเป็นตัวกำหนดว่าจะให้แปลงเลขจำนวนเต็มในตัวแปรออกมาเป็นเลขฐานใดบนหน้าจอ
* **`Serial.println(ตัวแปรทศนิยม, จำนวนตำแหน่ง);`**
  กำหนดทศนิยมที่ต้องการปริ้นท์ เช่น `Serial.println(myPi, 4);` จะปัดเศษและแสดงทศนิยม 4 ตำแหน่ง
